
# План миграции с Firebase на MySQL

Этот документ описывает процесс переноса бэкенда приложения SwiftMatch с облачной платформы Firebase на собственную связку с базой данных MySQL.

---

### Что было сделано

#### Часть 1: Перенос системы аутентификации

*   **Подготовлена база данных:** Создан SQL-файл `schema.sql` со структурой всех таблиц.
*   **Настроено окружение:** Установлены и настроены библиотеки для работы с MySQL, JWT, bcryptjs.
*   **Созданы API-маршруты:** Реализованы эндпоинты для регистрации (`/api/auth/register`) и входа (`/api/auth/login`).
*   **Обновлен UI:** Страницы регистрации и входа переведены на использование нового API.

#### Часть 2: Перенос функционала профиля пользователя

*   **Расширена схема БД:** Таблица `users` дополнена полями для хранения полной информации о профиле.
*   **Созданы API-маршруты:** Реализованы эндпоинты `GET /api/profile/me` и `PUT /api/profile`.
*   **Переписаны страницы профиля:** Страницы просмотра (`/profile`) и редактирования (`/profile/edit`) полностью переведены на работу с новым бэкендом.

#### Часть 3: Реализация загрузки фотографий

*   **Создан API для загрузки файлов:** Разработан API-маршрут `POST /api/upload`.
*   **Реализован UI для загрузки:** Создан компонент `PhotoUploader` и интегрирован в страницу редактирования профиля.

#### Часть 4: Миграция чатов и сообщений

*   **Созданы API для чатов:**
    *   `GET /api/chats` — получение списка всех чатов пользователя.
    *   `GET /api/chats/[chatId]/messages` — получение истории сообщений.
    *   `POST /api/chats/[chatId]/messages` — отправка нового сообщения.
*   **Переписан UI чатов:**
    *   Страница `src/app/chats/page.tsx` полностью переведена на новый API.
    *   Создана и реализована новая страница `src/app/chats/[chatId]/page.tsx`.

#### Часть 5: Миграция системы лайков и совпадений (Matches)

*   **Созданы API для лайков и поиска:**
    *   `GET /api/search/users` — получение списка потенциальных партнеров для свайпа.
    *   `POST /api/matches` — обработка лайка/дизлайка. Этот же эндпоинт отвечает за создание "совпадения" (match) и автоматическое создание чата между пользователями при взаимном лайке.
*   **Переписан UI поиска:**
    *   Страница поиска (`/search`) полностью переведена на новый бэкенд. Теперь она получает пользователей с сервера, отправляет действия и обрабатывает получение совпадений.

---

### Миграция завершена!

Весь основной функционал приложения был успешно перенесен с Firebase на собственный бэкенд с использованием Next.js, TypeScript и MySQL. Приложение полностью работоспособно и готово к дальнейшему развитию.
