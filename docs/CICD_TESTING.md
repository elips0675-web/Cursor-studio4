
# Тестирование и CI/CD в SwiftMatch

В проекте реализован профессиональный подход к обеспечению качества (QA) и автоматизации доставки кода (DevOps).

---

### 1. Unit-тестирование (Jest)

Мы используем Jest для проверки чистой логики приложения (алгоритмы скоринга, фильтры, утилиты).

*   **Команда:** `npm run test`
*   **Охват:** Покрыты критические функции в `src/lib/utils.ts` и `src/lib/word-filter.ts`.

---

### 2. End-to-End (E2E) Тестирование (Playwright)

Для гарантии того, что пользовательский интерфейс работает корректно во всех браузерах, внедрена структура для Playwright.

#### Основные сценарии (Test Suits):
1.  **Auth Flow:** Регистрация -> Вход -> Онбординг.
2.  **Matching Flow:** Открытие поиска -> Лайк -> Экран совпадения (Match).
3.  **Chat Flow:** Отправка сообщения -> Ответ AI-помощника.

*   **Запуск:** `npx playwright test`

---

### 3. Нагрузочное тестирование (K6)

Для подтверждения масштабируемости бэкенда (Firebase) подготовлены скрипты для K6.

*   **Цель:** Проверка работы Firestore при 10,000 одновременных соединений.
*   **Метрики:** Response time < 200ms для чтения профиля при высокой нагрузке.

---

### 4. CI/CD (GitHub Actions)

Процесс автоматизирован через GitHub Actions (`.github/workflows/ci.yml`):

1.  **Lint & Typecheck:** Проверка синтаксиса и типов TypeScript.
2.  **Run Tests:** Запуск Jest тестов.
3.  **Build:** Сборка Next.js проекта.
4.  **Deploy (Firebase App Hosting):** При успешном прохождении всех этапов, код автоматически деплоится в Google Cloud.

---

### 5. Мониторинг (Production)

*   **Sentry:** Сбор ошибок на стороне клиента (Runtime errors).
*   **Firebase Performance:** Мониторинг LCP и сетевых задержек у реальных пользователей.
*   **Google Cloud Dashboards:** Отслеживание таймаутов AI и ошибок базы данных.

---

**Этот конвейер гарантирует стабильность продукта при передаче новому владельцу.**
