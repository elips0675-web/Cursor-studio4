
# Тестирование и CI/CD в SwiftMatch

В проекте реализован современный подход к обеспечению качества и автоматизации доставки кода.

---

### 1. Тестирование (Testing)

Мы используем стандарт де-факто для React-приложений:

*   **Jest:** Мощный фреймворк для запуска тестов.
*   **React Testing Library:** Инструмент для тестирования компонентов с точки зрения пользователя (behavior-driven testing).

#### Как запустить тесты:
```bash
npm run test          # Одиночный запуск
npm run test:watch    # Режим разработки (автозапуск при изменениях)
```

#### Что мы тестируем:
*   **Unit-тесты:** Проверка чистых функций (например, расчет званий в `utils.ts`).
*   **Component-тесты:** Проверка корректности отрисовки UI-элементов и их реакции на действия пользователя.

---

### 2. CI/CD (Continuous Integration / Continuous Deployment)

Процесс автоматизирован для минимизации ошибок и быстрого выпуска обновлений.

#### GitHub Actions (CI)
При каждом пуше в ветку `main` запускается цепочка проверок:
1.  **Linting:** Проверка кода на соответствие стандартам (ESLint).
2.  **Type Checking:** Проверка корректности типов TypeScript.
3.  **Testing:** Запуск всех тестов Jest.

Если любая из проверок не проходит, сборка считается неудачной, что предотвращает попадание багов в продакшн.

#### Firebase App Hosting (CD)
Для автоматического развертывания используется **Firebase App Hosting**:
*   **Авто-деплой:** После успешного прохождения всех проверок в GitHub Actions, Firebase автоматически собирает и публикует новую версию приложения.
*   **Управление окружением:** Переменные среды (`.env`) настраиваются в консоли Firebase.
*   **Масштабируемость:** Google Cloud автоматически управляет мощностями сервера в зависимости от нагрузки.

---

### 3. Дальнейшие шаги
Для полноценного продукта рекомендуется добавить:
*   **E2E тесты (Playwright):** Для тестирования полных сценариев (регистрация -> свайп -> мэтч -> чат).
*   **Sentry:** Для мониторинга ошибок у реальных пользователей.
