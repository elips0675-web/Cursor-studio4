# Структура приложения SwiftMatch

Этот документ содержит подробное описание файловой структуры проекта и логику работы ключевых модулей.

## 📂 Обзор папок и файлов

### `/src` — Исходный код
Основная рабочая директория приложения.

#### `/src/app` — Маршрутизация и Страницы (Next.js App Router)
Здесь определяется структура URL и визуальное наполнение страниц.
*   `layout.tsx`: Корневой макет. Инициализирует шрифты, Firebase провайдеры, систему уведомлений (Toaster) и контексты (язык, фичи).
*   `page.tsx`: Главная страница. Содержит Hero-блок, ленту "Топ недели", баннер конкурса и популярные группы.
*   `login/` & `register/`: Страницы аутентификации.
*   `onboarding/`: Пошаговый процесс создания профиля с интеграцией AI для генерации био.
*   `search/`: Основной функционал знакомств. Поддерживает режимы "Люди рядом" и "Автопоиск".
*   `chats/`: Система обмена сообщениями. Разделена на личные диалоги (DMs) и тематические группы.
*   `user/`: Страница публичного профиля другого пользователя.
*   `profile/`: Личный кабинет пользователя, управление галереей и редактирование данных.
*   `groups/`: Каталог сообществ, разделенный по категориям (Музыка, Спорт, IT и т.д.).
*   `admin/`: Полноценная панель управления (Аналитика, Пользователи, Жалобы, Feature Flags, Контент).
*   `legal/`: Юридические страницы (Политика конфиденциальности, Условия использования).

#### `/src/components` — UI Компоненты
Разделены по ответственности для удобства поддержки.
*   `/ui`: Базовые примитивы ShadCN (Button, Input, Card, Dialog и др.).
*   `/layout`: Глобальные части интерфейса (AppHeader, BottomNav, AppContainer).
*   `/shared`: Общие элементы (CookieConsent, PwaInstallBanner, ZodiacIcon).
*   `/sections`: Крупные логические блоки (TopOfWeek, Recommendations).
*   `/dialogs`: Модальные окна (MatchDialog, PremiumDialog, AutosearchDialog).

#### `/src/context` — Глобальное состояние
*   `language-context.tsx`: Управляет переключением RU/EN во всем приложении.
*   `feature-flags-context.tsx`: Позволяет удаленно включать/выключать функции приложения через Firebase.

#### `/src/firebase` — Backend и База данных
*   `config.ts`: Ключи доступа к Firebase.
*   `provider.tsx`: React-контекст для доступа к Auth и Firestore.
*   `use-collection.tsx` & `use-doc.tsx`: Умные хуки для получения данных в реальном времени с автоматической подпиской.

#### `/src/lib` — Бизнес-логика и данные
*   `constants.ts`: Глобальные списки (интересы, цели, знаки зодиака).
*   `demo-data.ts`: База данных для демонстрационного режима (пользователи, группы).
*   `word-filter.ts`: Алгоритмы Anti-profanity (фильтрация мата и спама в чатах).
*   `utils.ts`: Вспомогательные функции (обработка CSS-классов, логика присвоения званий).

#### `/src/ai` — Искусственный интеллект (Genkit)
*   `/flows`: Логика AI-агентов (генератор био, генератор ледоколов для чата, анализ совместимости).
*   `genkit.ts`: Конфигурация модели Gemini.

## ⚙️ Ключевая логика

### 1. Алгоритм Автопоиска (`docs/AUTOSEARCH_LOGIC.md`)
Работает на системе Scoring (баллы). Пользователь получает высокий приоритет в выдаче, если его цели совпадают с вашими (+1000 баллов) или есть общие интересы (+100 за каждый).

### 2. Безопасность и Модерация
Любое сообщение в чате или группе проверяется через `word-filter.ts`. При обнаружении запрещенных слов или бессмысленного набора букв (gibberish), сообщение блокируется, а пользователю выводится предупреждение. Жалобы из профилей попадают напрямую в админку.

### 3. Real-time уведомления
В приложении реализована система "умных" уведомлений в `AppHeader`. Они отображают новые лайки, мэтчи и системные сообщения без перезагрузки страницы благодаря Firestore Listeners.

### 4. Система Званий (Ranks)
Автоматически рассчитывается в `utils.ts` на основе популярности анкеты, количества интересов и полноты био. Каждому пользователю присваивается одно наиболее статусное звание (например, "Король свиданий").
