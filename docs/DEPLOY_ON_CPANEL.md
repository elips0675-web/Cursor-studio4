# Как развернуть Next.js 15 (React 19) приложение на cPanel

Развертывание современного Node.js приложения на cPanel требует специфических шагов, так как cPanel — это в первую очередь среда для хостинга PHP-сайтов. Тем не менее, многие хостинг-провайдеры предоставляют инструменты для запуска Node.js.

**Важно:** Убедитесь, что ваш хостинг-план на cPanel включает в себя инструмент **"Setup Node.js App"** или аналогичный. Если его нет, развертывание будет невозможно.

---

### Шаг 1: Подготовка и сборка приложения

1.  **Установите зависимости:**
    Перед сборкой убедитесь, что все пакеты установлены. Выполните в терминале:
    ```bash
    npm install
    ```

2.  **Соберите проект для продакшена:**
    Next.js необходимо собрать в оптимизированный билд, который будет запускаться на сервере.
    ```bash
    npm run build
    ```
    Эта команда создаст папку `.next` с оптимизированным кодом.

### Шаг 2: Архивация и загрузка файлов

1.  **Создайте ZIP-архив.** Для загрузки на хостинг нужно запаковать только необходимые для запуска файлы и папки:
    *   `.next` (результат сборки)
    *   `public` (статические файлы: иконки, картинки)
    *   `node_modules` (установленные зависимости)
    *   `package.json`
    *   `package-lock.json`
    *   `next.config.mjs`

2.  **Загрузите архив на cPanel:**
    *   Войдите в вашу панель cPanel.
    *   Откройте **"Менеджер файлов" (File Manager)**.
    *   Создайте новую папку для вашего проекта (например, `swiftmatch-app`).
    *   Перейдите в созданную папку и загрузите ваш ZIP-архив.
    *   Распакуйте архив (правый клик -> Extract).

### Шаг 3: Настройка Node.js приложения в cPanel

1.  **Откройте "Setup Node.js App":**
    *   Вернитесь на главную страницу cPanel и найдите раздел **"Software"**.
    *   Откройте приложение **"Setup Node.js App"**.

2.  **Создайте новое приложение:**
    *   Нажмите кнопку **"Create Application"**.
    *   Заполните форму:
        *   **Node.js version:** Выберите последнюю доступную LTS-версию (например, 20.x.x).
        *   **Application mode:** `Production`.
        *   **Application root:** Укажите путь к папке, куда вы распаковали файлы (например, `swiftmatch-app`).
        *   **Application URL:** Выберите домен, на котором будет доступно приложение.
        *   **Application startup file:** Это самый важный пункт. Введите `node_modules/next/dist/bin/next`.

3.  **Добавьте переменные окружения:**
    *   Прокрутите вниз до раздела **"Environment Variables"**.
    *   Нажмите **"Add Variable"**.
    *   Введите ключ `GEMINI_API_KEY` и вставьте ваше значение.
    *   Добавьте также переменную для Next.js:
        *   Ключ: `NODE_ENV`, Значение: `production`

4.  **Сохраните приложение:**
    *   Нажмите кнопку **"Create"** вверху страницы.

### Шаг 4: Установка зависимостей и запуск

1.  После создания приложения вы вернетесь к списку ваших Node.js приложений.
2.  Найдите ваше новое приложение и нажмите **"Run NPM Install"**. Это установит зависимости из `package.json` в окружении сервера. **Не пропускайте этот шаг!**
3.  После успешной установки зависимостей нажмите **"Start App"**.

### Шаг 5: Настройка перенаправления (Port Forwarding)

Node.js приложение запускается на внутреннем порту. Чтобы сделать его доступным по вашему доменному имени, нужно настроить файл `.htaccess`.

1.  **Откройте "Менеджер файлов"** и перейдите в корневую папку вашего домена (`public_html` или папка с именем домена).
2.  **Создайте или отредактируйте файл `.htaccess`**.
3.  **Добавьте следующий код:** Замените `YOUR_APP_PORT` на порт, который был назначен вашему приложению (его можно увидеть в интерфейсе "Setup Node.js App").

    ```apache
    RewriteEngine On
    RewriteCond %{REQUEST_FILENAME} !-f
    RewriteCond %{REQUEST_FILENAME} !-d
    RewriteRule ^(.*)$ http://127.0.0.1:YOUR_APP_PORT/$1 [P,L]
    ```

---

После выполнения этих шагов ваше Next.js приложение должно быть доступно по указанному URL. Если возникают ошибки, проверьте логи в интерфейсе "Setup Node.js App".
